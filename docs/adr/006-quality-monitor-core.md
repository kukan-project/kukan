# ADR-006: Quality Monitor をコア機能として組み込む

## ステータス
承認済み（2026-03-01）

## コンテキスト
CKANにはデータ品質の監視機能がなく、リンク切れや不正CSVが放置される問題がある。
外部ツール（CKAN QA extension等）は存在するが、導入率が低く、運用が分離してしまう。
KUKANでは「品質はカタログの責任」という思想で、品質監視をコア機能に含める。

## 決定
Quality Monitor を `packages/quality` としてコアに組み込む。
外部プラグインではなく、カタログ本体の一部として動作する。

## 品質チェッカー（Phase 4で実装）

| チェッカー | 機能 | 頻度 |
|-----------|------|------|
| LinkChecker | リソースURLの死活監視 | 日次 |
| CsvValidator | CSV構造検証（ヘッダー、型、エンコーディング） | アップロード時 + 週次 |
| MetadataAuditor | 必須メタデータの充足率スコアリング | リアルタイム |
| PiiScanner | 個人情報の検出（AIAdapter利用） | アップロード時 |

## 品質スコア
- 各データセットに 0-100 のスコアを付与
- ダッシュボードで組織別・カテゴリ別の品質推移を可視化
- CKAN互換APIでスコアを外部公開可能

## 根拠
- 自治体ポータルでは「登録したら終わり」になりがち → 品質劣化が最大の課題
- コア機能なら全インスタンスで品質監視が有効 → エコシステム全体の品質向上
- Quality Monitor（検知）+ Data Editor（修正）で完全な品質ライフサイクル

## 影響
- `packages/quality` に品質チェッカーインターフェースと実装を配置
- Ingestパイプラインの最終ステップとして品質チェックを実行
- DBに `quality_score` テーブルを追加（データセット・リソース単位）
- ダッシュボードUIは `apps/web` 内に品質タブとして実装
