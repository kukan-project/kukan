## デプロイ方法

KUKAN は2つのデプロイ方式に対応しています。

### AWS デプロイ

AWS CDK を使って ECS Fargate + ALB 構成でデプロイします。

```bash
cd infra
npx cdk deploy
```

主要な AWS リソース:

- **ECS Fargate** — Web アプリケーション + Pipeline Worker
- **ALB** — ロードバランサー
- **Aurora Serverless v2** — PostgreSQL データベース
- **S3** — ファイルストレージ
- **SQS** — パイプラインキュー
- **OpenSearch** — 全文検索エンジン

### Docker Compose（オンプレミス）

Docker Compose を使ったオンプレミス・閉域網デプロイに対応しています。

```bash
cp .env.prod.example .env.prod
# .env.prod を編集
docker compose --env-file .env --env-file .env.prod --profile prod up -d --build
```

オンプレミス構成では以下の代替サービスを使用します。

| AWS サービス | オンプレミス代替 |
|-------------|----------------|
| S3 | MinIO |
| SQS | ElasticMQ |
| OpenSearch | PostgreSQL 全文検索 |

## ユーザー管理

システム管理者（sysadmin）は管理画面からユーザーを管理できます。

### ユーザー一覧・編集

- ユーザーの表示名・役割（user / sysadmin）の変更
- 自分自身の役割降格は不可（ロックアウト防止）

### ユーザーの削除

- ソフトデリート方式（state を deleted に変更）
- 削除時に全セッション・API トークンを即時無効化
- 監査ログ・アクティビティ履歴は保持

### 自己登録制御

環境変数 `REGISTRATION_ENABLED` で新規ユーザーの自己登録を制御します（デフォルト: 有効）。

## ヘルスチェック

外部 URL リソースの死活監視を自動で行います。

- 定期的なスケジューラーによる自動チェック
- 管理画面からヘルスチェック結果の確認
- エラーリソースの特定と対応

## 環境変数

主要な環境変数の一覧です。詳細は `.env.example` を参照してください。

| 変数名 | 説明 | デフォルト |
|--------|------|-----------|
| `POSTGRES_HOST` | PostgreSQL ホスト | `localhost` |
| `POSTGRES_PORT` | PostgreSQL ポート | `5432` |
| `POSTGRES_DB` | データベース名 | `kukan` |
| `POSTGRES_USER` | データベースユーザー | `kukan` |
| `POSTGRES_PASSWORD` | データベースパスワード | `kukan` |
| `S3_BUCKET` | S3 バケット名 | `kukan-dev` |
| `S3_ENDPOINT` | S3 エンドポイント（MinIO: `http://localhost:9000`、AWS: 省略） | - |
| `SEARCH_TYPE` | 検索エンジン（`opensearch` / `postgres`） | `opensearch` |
| `OPENSEARCH_URL` | OpenSearch エンドポイント | `http://localhost:9200` |
| `SQS_QUEUE_URL` | SQS キュー URL | - |
| `SQS_ENDPOINT` | SQS エンドポイント（ElasticMQ: `http://localhost:9324`、AWS: 省略） | - |
| `BETTER_AUTH_SECRET` | 認証セッション秘密鍵（32 文字以上） | - |
| `BETTER_AUTH_URL` | 公開 URL | `http://localhost:3000` |
| `REGISTRATION_ENABLED` | ユーザー自己登録の有効/無効 | `true` |
| `LOG_LEVEL` | ログレベル（`trace`〜`fatal`） | `info` |
