# Security Policy / セキュリティポリシー

## Supported Versions / サポート対象バージョン

KUKAN is currently in beta. Security fixes are applied to the latest
development branch only.

KUKAN は現在ベータ版です。セキュリティ修正は最新の開発ブランチにのみ
適用されます。

| Version   | Supported          |
| --------- | ------------------ |
| `develop` | :white_check_mark: |

## Reporting a Vulnerability / 脆弱性の報告

**Do not open a public GitHub issue for security vulnerabilities.**

**セキュリティ脆弱性について、公開の GitHub Issue を作成しないでください。**

Please report vulnerabilities by emailing **root@info-lounge.jp**.
Include the following information:

脆弱性は **root@info-lounge.jp** にメールで報告してください。
以下の情報を含めてください:

- Description of the vulnerability / 脆弱性の説明
- Steps to reproduce / 再現手順
- Potential impact / 想定される影響
- Suggested fix (if any) / 修正の提案（もしあれば）

## Response Timeline / 対応スケジュール

- **Acknowledgment / 受領確認**: Within 3 business days / 3 営業日以内
- **Initial assessment / 初期評価**: Within 10 business days / 10 営業日以内
- **Fix release / 修正リリース**: Depends on severity / 深刻度に応じて対応

## Disclosure Policy / 開示ポリシー

We follow coordinated disclosure. We will work with you to understand
and address the issue before any public disclosure.

協調的開示の方針を取ります。公開前に、報告者と協力して問題の理解と
対処を行います。

## Scope / 対象範囲

This policy applies to the KUKAN repository and its official
deployments. Third-party plugins or forks are not covered.

本ポリシーは KUKAN リポジトリおよびその公式デプロイメントに適用されます。
サードパーティのプラグインやフォークは対象外です。
